# 📝 Task Tracker REST API

RESTful API сервис для управления задачами с полноценной системой безопасности и разделением ролей. Проект построен на классической многослойной архитектуре с использованием современных стандартов аутентификации.

## 🛠 Технологический стек
* **Java 17+**
* **Spring Boot 3** (Web, Data JPA)
* **Spring Security 6** + **JWT (JSON Web Tokens)**
* **PostgreSQL** (хранение данных пользователей и задач)
* **Swagger / OpenAPI 3** (автоматическая документация API)
* **Maven** (сборка проекта)

## Архитектура Безопасности & Изоляция Данных

В проекте реализована строгая привязка данных к авторизованному пользователю:
* **Сущности (Entities):** Между таблицами `users` (PostgreSQL) и `tasks` настроена связь `@ManyToOne` (каждая задача строго принадлежит одному пользователю).
* **Создание (POST):** Идентификатор `user_id` не передается в JSON-теле. Сервер автоматически извлекает имя пользователя из JWT-токена, находит его в БД и привязывает к новой задаче перед сохранением.
* **Модификация и удаление (PUT/DELETE):** Доступ к изменению или удалению задачи возможен только в том случае, если имя пользователя из JWT-токена совпадает с владельцем (`user_id`) этой задачи в базе данных. При попытке изменить чужую запись возвращается статус `403 Forbidden`.
### 🔐 Безопасность и Авторизация (Spring Security)
В проекте реализована гибридная система безопасности:
* **Stateless аутентификация через JWT:** Сервер не хранит сессии, доступ к защищенным эндпоинтам осуществляется через Bearer-токены.
* **Хэширование паролей:** Все пароли пользователей перед сохранением в БД надежно шифруются с помощью алгоритма `BCrypt`.
* **Гибридная авторизация:** Реализован кастомный `UserDetailsService`, который поддерживает два типа пользователей:
  * Обычные пользователи (загружаются из PostgreSQL).
  * Вшитый в оперативную память `superadmin` (защита от компрометации БД).
* **Role-Based Access Control (RBAC):** Маршрутизация и права доступа зависят от роли пользователя (например, `ROLE_USER`, `ROLE_ADMIN`).

### ⚙️ Бизнес-логика и Архитектура
* **Слоистая архитектура:** Четкое разделение ответственности между контроллерами (обработка HTTP), сервисами (бизнес-логика) и репозиториями (работа с БД).
* **Паттерн DTO (Data Transfer Object):** Изоляция внутренних сущностей базы данных (Entity) от внешних клиентов. Данные передаются через специализированные Record-классы.
* **CRUD операции для задач:** Создание, чтение, обновление и удаление задач с привязкой к конкретному пользователю.

## 📄 Документация API (Swagger)
В проекте настроена автоматическая генерация документации. После запуска приложения интерактивный интерфейс для тестирования всех запросов (с поддержкой передачи JWT-токена) доступен по адресу:
`http://localhost:8080/swagger-ui/index.html`

## 🏃‍♂️ Как запустить локально

1. Склонируйте репозиторий:
   ```bash
   git clone [https://github.com/твой-ник/task-tracker.git](https://github.com/твой-ник/task-tracker.git)
2. Настройте подключение к PostgreSQL в файле src/main/resources/application.properties:
   ```
   spring.datasource.url=jdbc:postgresql://localhost:5432/твоя_бд
   spring.datasource.username=твой_логин
   spring.datasource.password=твой_пароль
   ```
 3. Запустите проект через Maven или вашу IDE. Таблицы в базе данных сгенерируются автоматически благодаря Hibernate.
